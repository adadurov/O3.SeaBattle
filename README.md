# Веб сервис "Морской бой"

## Ограничения и замечания

* Аутентификация и авторизация не требуется.

* HTTPS не поддерживается.

* Есть swagger. При запуске приложения браузер направляется на /swagger и открывается документация.

* В ТЗ странные соглашения об именовани свойств в моделях для запросов и ответов. Реализовано как просили. :)

* Размер акватории (квадратной матрицы) не более 26, т.к. из ТЗ не понятен принцип адресации для матрицы бОльшего размера.

* При инциализации игры производится поиск перекрывающихся кораблей. Алгоритм простоейший, сложность O(N*N).
  В реальной задаче стоит использовать структуру для spatial index типа R-Tree.

* Поиск подбитых кораблей ведется последовательным перебором, медленный. R-Tree и здесь должно помочь.

* Сервис не сохраняет состояние игры в персистентном хранилище, поэтому при перезапуске сервиса состояние игры теряется.

## Замечания об API

* /ship -- возвращает 400, если корабли пересекаются. Допускается касание кораблей.
* /state -- возвращает 400, если клиент пытается запросить статистику до начала игры
* /shot -- возвращает 400, если клиент стреляет за пределы матрицы
